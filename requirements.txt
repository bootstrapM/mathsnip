# Local OCR engine (Pix2Text). Its heavy ML dependencies (torch, transformers,
# tokenizers, safetensors, numpy, opencv…) are intentionally left UNPINNED so
# each machine installs the current, internally-consistent set — which is what
# works. The Pix2Text model needs a *recent* transformers/torch, so do NOT pin
# these down.
#
# The HF/torch stack must be pinned as ONE matched set from the torch 2.4 era
# (~late 2024). Each constraint forces the next:
#   * optimum<2          — 2.0 relocated `main_export` (breaks Pix2Text's import)
#   * optimum 1.x        — imports `_attention_scale`, REMOVED in torch >=2.5  -> torch<2.5
#   * model tokenizer    — needs recent tokenizers -> transformers>=4.44
#   * transformers/safetensors after ~4.47 call `torch.uint16`/`torch.int4`
#     (need torch>=2.6) -> cap transformers<4.47 and safetensors<0.5
#   * torch>=2.4         — provides torch.uint16 that transformers 4.44-4.46 use
# Net: torch 2.4.x + transformers 4.44-4.46 + safetensors 0.4.x + optimum 1.x.
# (Apple Silicon only — torch 2.4 has no macOS Intel wheels; Intel = cloud engine.)
torch>=2.4,<2.5
torchvision>=0.19,<0.20
transformers>=4.44,<4.47
safetensors>=0.4.3,<0.5
optimum>=1.20,<2
# onnxruntime >=1.20 hard-references torch.int4 (only in torch 2.6+) in its
# io_binding_helper, breaking model load on torch 2.4. 1.19.x predates it.
onnxruntime<1.20
pix2text>=1.1.4
Pillow>=10.0

# LaTeX -> MathML (used for the MathML / Word tab)
latex2mathml>=3.77

# Global hotkey listener
pynput>=1.7

# Cloud engine (OpenAI-compatible vision APIs).
requests>=2.31
